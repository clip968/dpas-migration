## 빌드 선행 조건 설치

현재 호스트는 macOS ARM 환경이다. macOS 기본 도구만으로는 Linux 커널 빌드 요구사항을 만족하지 못했다.

- macOS 기본 `make`는 GNU Make 3.81이라 커널 요구사항인 4.0 이상에 부족했다.
- macOS 기본 `bash`는 3.2라 커널 요구사항인 4.2 이상에 부족했다.
- `brew`는 `/opt/homebrew/bin/brew`에 이미 있었지만 PATH에는 잡혀 있지 않았다.
- `colima`, `docker`, `gmake`, `pahole`, `ld.lld` 등은 처음에는 PATH에 없었다.

선택한 설치 방향:

- Mac ARM에서 직접 커널 빌드를 억지로 맞추지 않고, Colima/Docker 기반 Ubuntu 24.04 Linux 빌드 이미지 안에서 `dpas-kernel`을 빌드하는 방향으로 준비했다.
- Homebrew로 `colima 0.10.3`, `docker 29.5.3`을 설치했다.
- Colima VM은 CPU 4개, 메모리 8GiB, 디스크 80GiB로 시작했다.
- Colima VM에 `/Volumes/CodeCS/dpas-migration`을 writable mount로 추가했다.

Docker 빌드 이미지:

- 이미지 이름: `dpas-kernel-build-env:ubuntu24.04`
- 기반 이미지: `ubuntu:24.04`
- 주요 패키지:
  - `build-essential`, `make`, `bash`, `bc`
  - `bison`, `flex`, `gawk`
  - `libssl-dev`, `openssl`
  - `libelf-dev`, `libdw-dev`, `dwarves`, `pahole`
  - `rsync`, `cpio`, `xz-utils`, `zstd`
  - `gcc-x86-64-linux-gnu`, `binutils-x86-64-linux-gnu`
  - `clang`, `llvm`, `lld`
  - `python3`, `perl`, `pkg-config`, `git`

## 헷갈렸던 점

- 첫 Docker mount 테스트는 마지막 `id` 명령이 성공해서 전체가 성공한 것처럼 보였지만, 실제로는 `test -f dpas-kernel/Makefile` 실패를 놓쳤다.
- 원인은 Colima VM에 `/Volumes/CodeCS/dpas-migration`이 mount되어 있지 않아 Docker가 빈 `/work` 디렉터리를 만든 것이었다.
- Colima를 stop/start하고 `--mount /Volumes/CodeCS/dpas-migration:w`를 추가한 뒤 해결됐다.

## 남은 리스크와 다음 작업

- 오늘 검증은 compile/link와 `bzImage` 생성까지다.
- 새 `bzImage`를 VM에 올려 실제 부팅하거나 DPAS sysfs/runtime I/O를 검증하지는 않았다.
- 다음 작업은 `build/dpas-kernel-vm/arch/x86/boot/bzImage`를 기존 VM 재현 흐름에 연결해 부팅 확인을 하는 것이다.

## Colima VM 제어 래퍼 추가

반복해서 긴 Colima 명령을 치지 않도록 top-level에 `vm` 실행 스크립트를 추가했다.

- `./vm start`: CPU 4개, 메모리 8GiB, 디스크 80GiB, 현재 repo writable mount로 Colima 시작.
- `./vm stop`: Colima 중지.
- `./vm status`: 현재 Colima 상태 확인.
- `./vm ssh`: Colima가 이미 켜져 있을 때만 VM에 SSH 접속. 꺼져 있으면 자동 시작하지 않고 `./vm start` 안내 후 실패.

검증:

- `sh -n vm` 통과.
- `./vm status`가 현재 실행 중인 Colima 상태를 정상 출력했다.
- Colima가 꺼진 상태에서 `./vm ssh -- true`는 자동 시작 없이 `error: colima is not running`으로 실패했다.
