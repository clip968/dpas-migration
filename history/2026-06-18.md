## dpas-kernel clangd 선택 DB 생성

사용자 요청에 따라 `dpas-kernel` git history에서 실제 DPAS 작업에 등장한 커널 C 파일만 골라 clangd용 DB를 만들었다.

- 기준 이력: `dpas-kernel` root import commit 이후 현재 `HEAD`까지.
- 선택된 C 파일:
  - `block/blk-core.c`
  - `block/blk-mq.c`
  - `block/blk-sysfs.c`
  - `block/fops.c`
  - `block/blk-stat.c`
  - `fs/iomap/direct-io.c`
- 처음에는 C 파일 6개만 DB에 넣었다.
- 이후 사용자가 header 파일도 적용되어야 한다고 지적해서 history에 나온 기존 `.h` 파일 4개도 명시 entry로 추가했다.
- header는 kbuild가 `.o`로 직접 컴파일하지 않으므로 가까운 C 파일의 kbuild command를 빌려 `-x c-header`로 파싱하게 했다.
- 삭제된 `block/blk-dpas.c`와 Python selftest 파일은 clangd C DB 대상에서 제외했다.

추가한 helper:

- top-level `./clangd-db`
- `scripts/dpas_clangd_db.py`
- `scripts/test_dpas_clangd_db.py`
- `dpas-kernel/.clangd`

생성 결과:

- `dpas-kernel/compile_commands.json`
- entry 수: 10개
- 포함 header:
  - `include/linux/blkdev.h`
  - `include/linux/blk_types.h`
  - `block/blk-mq.h`
  - `block/blk-stat.h`
- `/work/...` 컨테이너 경로는 host 경로 `/Volumes/CodeCS/dpas-migration/...`로 치환됐다.
- `directory`는 out-of-tree build dir인 `build/dpas-kernel-vm`를 가리킨다.

검증:

- `python3 -m unittest scripts/test_dpas_clangd_db.py` 통과.
- `python3 -m py_compile scripts/dpas_clangd_db.py scripts/test_dpas_clangd_db.py` 통과.
- `sh -n clangd-db` 통과.
- `./clangd-db --print-files`가 C 파일 6개와 header 4개, 총 10개 entry를 재생성했다.
- `python3 -m json.tool dpas-kernel/compile_commands.json` 통과.
- `rg -n "/work" dpas-kernel/compile_commands.json dpas-kernel/.clangd`는 매치 없음.
- `clangd --check`는 `compile_commands.json`과 `.clangd`를 읽고 preamble/AST를 만들었다. 다만 Apple clangd check mode가 Linux kernel 매크로에서 `SwapBinaryOperands` tweak replacement overlap을 보고해 exit 3으로 끝난다. 이는 DB 발견 실패나 include 경로 실패가 아니라 check mode의 전체-token feature 테스트 쪽 문제다.
- `clangd --check include/linux/blkdev.h`도 CDB의 header entry를 읽고 `-x c-header`로 preamble/AST를 만들었다. 다만 header 전체-token check에서도 Apple clangd가 URI/tweak 관련 오류를 많이 출력해 exit 3으로 끝난다.

주의:

- `dpas-kernel/.gitignore`가 `.*`와 `compile_commands.json`을 ignore하므로 `dpas-kernel/.clangd`와 `dpas-kernel/compile_commands.json`은 해당 repo의 `git status`에 표시되지 않는다.
- 새로 DPAS 관련 C 파일을 작업하면 `./clangd-db`를 다시 돌리면 된다. `.cmd`가 없는 파일이면 먼저 해당 `.o`만 kbuild로 만들고 다시 실행해야 한다.
