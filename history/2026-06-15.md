- `dpas-kernel`의 full DPAS mode-switch 포팅 코드는 아직 커밋되지 않은 중간 상태다.

### 이어받은 문제

- `dpas-kernel/include/linux/blkdev.h`의 `enum dpas_mode`가 `struct request_queue` 내부에 타입 선언만 들어가 있어 `-Werror` 빌드에서 실패한다.

- `switch_param1~7`의 허용 범위와 signed/unsigned 비교를 정리한다.

## full DPAS mode switching 테스트 재작성

사용자가 기존 static test를 전면 폐기하고, 현재 코드의 full DPAS mode switching을 검증하는 새 테스트 코드를 요청했다.

변경:

- 기존 테스트 4개를 삭제했다.
  - `no_q_dpas_static.py`
  - `pas_queue_lifecycle_static.py`
  - `pas_sysfs_static.py`
  - `request_queue_pas_stat_static.py`
- 새 테스트 `dpas-kernel/tools/testing/selftests/dpas/full_mode_switching_static.py`를 추가했다.

새 테스트가 검증하는 항목:

- `request_queue` 기반 DPAS 상태 필드와 `enum dpas_mode`.
- queue 초기화 시 `switch_enabled`, mode, counter, QD/tf 초기값.
- `switch_enabled` sysfs show/store와 reset 경로.
- `blk_dpas_prepare_bio()`의 submit-time gate.
  - DPAS off이면 기존 HIPRI polling 유지.
  - INT이면 `IOCB_HIPRI`/`REQ_POLLED` 제거.
  - `INT->OL`은 submit helper에서 수행.
  - CP/PAS/OL은 polled 유지와 mode별 counter 증가.
- `blk_dpas_maybe_switch_mode()`의 poll-time 전이표.
  - `CP->PAS`
  - `PAS->OL`
  - `PAS->CP`
  - `PAS` 유지
  - `OL->PAS`
  - `OL->INT`
  - `OL` 유지
  - `INT`는 poll path에서 전이하지 않음.
- PAS duration clamp 시 `tf` 증가, PAS sleep 전후 QD 표본 누적, completion 후 mode 평가 호출.

검증:

- `full_mode_switching_static.py` 통과.
- `python3 -m py_compile full_mode_switching_static.py` 통과.
- `git diff --check` 통과.
- 새 파일 no-index whitespace check에서 whitespace 오류 없음.
- `/tmp/dpas-build` out-of-tree 변경 object 빌드 통과.

## DPAS 학습 노트 기반 동작 시나리오 정리

사용자가 `DPAS-학습-노트.md`를 기반으로 현재 코드가 실제로 어떻게 동작하는지 시나리오 형태의 상세 설명을 요청했다.

정리한 핵심 흐름:

- `pas_enabled=1`, `switch_enabled=1`, HIPRI direct I/O를 전제로 한다.
- sysfs에서 `switch_enabled`를 쓰면 mode/counter/window가 PAS 기준으로 reset된다.
- submit path의 `blk_dpas_prepare_bio()`가 현재 mode에 따라 `REQ_POLLED`를 켜거나 끄고 mode별 submit counter를 올린다.
- poll path의 `blk_mq_poll_pas_sleep()`이 PAS/OL QD 표본과 PAS duration clamp 횟수(`dpas_tf`)를 모은다.
- completion 후 `blk_dpas_maybe_switch_mode()`가 `PAS->CP`, `PAS->OL`, `CP->PAS`, `OL->PAS`, `OL->INT`를 판단한다.
- `INT->OL`은 INT I/O가 poll path를 타지 않기 때문에 submit helper에서만 처리된다.
